Networking Basics

⬡
Networking = connecting devices so they can share data.

Think of it like building roads between houses — data travels the roads instead of people.

◇
Two essential terms to remember:
  • Node: Any device on the network — your phone, a laptop, a printer, a server.
  • Communication Link: The path data travels — Wi-Fi waves or physical cables.

02

Devices in a Network

Quick rule: End devices use data. Intermediary devices move data.

◫ End Devices

Where data starts and stops its journey.

  • Desktop / Laptop
  • Mobile Phone / Tablet
  • Printer / Smart TV
  • Server

⇄ Intermediary Devices

The traffic directors of your network.

⬡ Switch

Connects devices in the same building (LAN). Knows exactly which port to send data to, minimizing traffic.

⟿ Router

Connects your local network to the internet — like a post office that forwards letters to the right city.

◎ Modem

Translates internet signals (from your ISP) into a format your router understands, and vice versa.

03

Network Characteristics

The four pillars that define how reliable, scalable, and secure a network is.

♡

Fault Tolerance

If one cable breaks, the network reroutes automatically. Like a GPS finding an alternate path when the road is closed.

⤢

Scalability

Add 1,000 new devices without slowing down existing ones. Good architecture grows without crumbling.

◉

Quality of Service (QoS)

VIP treatment for video calls — they get priority bandwidth so they don't stutter when someone downloads a large file.

⊛

Security (CIA Triad)

Protects data via Confidentiality (who sees it), Integrity (isn't altered), and Availability (always accessible).

05

Protocols — The Rules

⊟

A protocol governs What is communicated, How it's formatted, and When it's sent — like a language both devices agree to speak.

◈ Protocol Elements

  • Encoding: Converting text/video into digital signals that travel over cables or air.
  • Encapsulation: Wrapping data in digital envelopes with addressing info.
  • Message Size: Breaking large files into manageable chunks called packets.
  • Timing: Rules for when to send and how fast — avoiding data collisions.

⇶ Delivery Methods

  • Unicast: Sending to exactly one specific device — like a direct message.
  • Multicast: Sending to a selected group — like a group chat.
  • Broadcast: Shouting to every device on the network — like a PA system.
